# Accessibility Skill

Use this skill to enforce WCAG 2.1 AA practices in frontend features.

## Accessibility baseline
- Semantic markup and heading hierarchy
- Labeling for form controls
- Keyboard operability
- Visible focus states
- Sufficient color contrast
- Announce dynamic status updates where needed

## Testing guidance
- Prefer `getByRole` and accessible name assertions.
- Include tab-flow checks for interactive views.
- Validate aria relationships for composite widgets.
